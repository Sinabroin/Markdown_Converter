File-to-Markdown

다양한 문서를 LLM·RAG 전처리에 바로 쓸 수 있는 정규화된 Markdown으로 변환합니다. 자동 라우팅, 품질 검증, 에스컬레이션, JSON 로깅이 모두 포함된 프로덕션 파이프라인입니다.

Python 3.13 Windows 10/11 MarkItDown 0.1.5 PowerShell / CMD 14개 포맷 지원 🆓 PaddleOCR 무료 이미지 OCR

🔍 개요

File-to-Markdown은 Microsoft의 MarkItDown을 변환 엔진으로 사용하고, 그 위에 라우팅·정규화·검증·로깅 파이프라인을 감싼 Windows CLI 도구입니다. PDF, DOCX, PPTX, XLSX, HTML 등 다양한 파일을 입력하면 정제된 Markdown 파일과 변환 로그를 자동으로 생성합니다.

🗺️

자동 라우팅

확장자·MIME 기반으로 최적 변환 모드를 자동 선택합니다.

🪜

4단계 에스컬레이션

Standard → PaddleOCR → OCR → High-accuracy 순으로 자동 품질 보장합니다.

✨

Markdown 정규화

공백·불릿·헤딩·반복 토큰을 자동으로 정리합니다.

🔎

품질 검증

최소 길이·추출 비율·반복 이상 등 5개 항목을 자동 검사합니다.

📋

JSON 로깅

변환 건마다 모드·시간·검증 결과를 JSON으로 기록합니다.

⚡

배치 처리

폴더 단위 병렬 변환과 진행률·실패 리포트를 지원합니다.

📦 설치

Python 3.13 이상이 필요합니다. 가상환경 사용을 강력히 권장합니다.

  1. 1

    프로젝트 폴더로 이동

    powershell 
    cd D:\Projects\File-to-markdown
  2. 2

    가상환경 생성 및 활성화

    powershell 
    python -m venv .venv
.venv\Scripts\activate

    활성화되면 프롬프트 앞에 (.venv)가 표시됩니다.

  3. 3

    의존성 설치

    powershell 
    pip install -r requirements.txt

    markitdown, pyyaml, magika 등 모든 의존성이 자동으로 설치됩니다 (약 2~5분 소요).

  4. 4

    설치 확인

    powershell 
    python -c "from markitdown import MarkItDown; print('OK')"

    OK가 출력되면 설치 완료입니다.

  5. 5

    PaddleOCR 설치 이미지 변환 시 필요

    powershell 
    pip install paddlepaddle paddleocr

    .jpg, .png, .tif 등 이미지 파일을 변환할 때만 필요합니다. PDF·Word 등 일반 문서 변환에는 설치하지 않아도 됩니다.

    ⚠️
    첫 실행 시 한국어 모델이 자동 다운로드됩니다 (약 500MB~1GB). 인터넷 연결이 필요하며 최초 1회만 진행됩니다. 이후 실행부터는 즉시 시작됩니다.

🚀 빠른 시작

변환하고 싶은 파일을 input\ 폴더에 넣거나, 파일 경로를 직접 지정합니다.

powershell — 가장 간단한 사용법 
# input/ 폴더에 넣은 파일 변환
python scripts\convert_one.py input\문서.pdf

# 파일 경로를 직접 지정
python scripts\convert_one.py "C:\Users\내폴더\보고서.docx"

변환이 완료되면 아래와 같이 출력됩니다:

출력 예시
[OK] 문서.pdf → 문서.md  (12.34s, standard)

결과 파일은 다음 위치에 저장됩니다:

output\
├── raw_md\ ← MarkItDown 원본 출력 (보존용)
├── clean_md\ ← 최종 정규화 완료 (실제 사용할 파일)
├── json_logs\ ← 변환 로그 JSON
└── failed\ ← 검증 실패 파일 (있을 경우)
💡
실제로 쓸 파일은 output\clean_md\ 안에 있습니다. raw_md\는 원본 보존 목적으로만 유지됩니다.

📄 단일 파일 변환

scripts\convert_one.py는 파일 1개에 대해 전체 파이프라인(추출→정규화→검증→로그)을 실행합니다.

기본 사용법

powershell 
python scripts\convert_one.py <파일경로> [옵션]

옵션

옵션설명예시
--force 이미 변환된 파일이 있어도 재변환합니다 --force
--mode 변환 모드를 강제 지정합니다 (기본: 자동) --mode ocr
--config 설정 파일 경로를 지정합니다 --config my\settings.yaml

사용 예시

powershell 
# 기본 변환 (PDF, DOCX 등)
python scripts\convert_one.py input\report.pdf

# 이미지 변환 (PaddleOCR 자동 사용 — API 키 불필요)
python scripts\convert_one.py input\scan.jpg
python scripts\convert_one.py input\invoice.png

# PaddleOCR 모드로 강제 지정
python scripts\convert_one.py input\scan.jpg --mode paddle_ocr

# 재변환 (기존 결과 덮어쓰기)
python scripts\convert_one.py input\report.pdf --force

# OCR 모드로 강제 지정 (OpenAI 키 필요)
python scripts\convert_one.py input\scan.pdf --mode ocr

# 절대 경로 사용 — input 폴더 불필요
python scripts\convert_one.py "C:\Users\user\Desktop\계약서.docx"
python scripts\convert_one.py "D:\스캔이미지\영수증.png"
ℹ️
이미 변환한 파일을 다시 변환하려면 --force 옵션을 추가하세요. 기본적으로 raw_md\에 같은 이름 파일이 있으면 변환을 건너뜁니다.

📂 배치 변환

폴더 안의 파일을 한 번에 변환합니다. scripts\convert_batch.py는 병렬 처리와 진행률 표시, 완료 리포트를 지원합니다.

기본 사용법

powershell 
python scripts\convert_batch.py <폴더경로> [옵션]

옵션

옵션설명기본값
--recursive 하위 폴더까지 포함합니다 미포함
--workers 병렬 변환 워커 수 4
--force 이미 변환된 파일도 재변환 미사용

사용 예시

powershell 
# input/ 폴더 전체 변환
python scripts\convert_batch.py input\

# 하위 폴더 포함, 병렬 8개
python scripts\convert_batch.py input\ --recursive --workers 8

# 전체 재변환
python scripts\convert_batch.py input\ --recursive --force

✨ 정규화 단독 실행

이미 변환된 raw_md\ 파일을 다시 정규화하고 싶을 때 사용합니다.

powershell 
# raw_md의 파일을 정규화 → clean_md/에 저장
python scripts\normalize_md.py output\raw_md\report.md

# 출력 경로 지정
python scripts\normalize_md.py output\raw_md\report.md -o output\clean_md\report_v2.md

# 결과를 화면에 출력 (파일 저장 안 함)
python scripts\normalize_md.py output\raw_md\report.md --stdout

정규화 처리 내용

처리 항목설명
후행 공백 제거각 줄 끝의 불필요한 공백을 제거
연속 개행 축소3줄 이상 연속 빈 줄 → 2줄로 축소
불릿 통일*, + 불릿 → -로 통일
빈 헤딩 제거내용 없는 ## 줄 삭제
반복 토큰 축소10자+ 동일 문자열 3회+ 반복 → 1회로 축소
코드블록 보호코드블록 내부 내용은 변경하지 않음

🔎 검증 단독 실행

변환된 Markdown 파일의 품질을 수동으로 검사합니다.

powershell 
# 기본 검증
python scripts\validate_md.py output\clean_md\report.md

# 원본 파일 크기를 지정해 추출 비율도 검사
python scripts\validate_md.py output\clean_md\report.md --input-size 204800

# JSON 형식으로 결과 출력
python scripts\validate_md.py output\clean_md\report.md --json

검증 항목

항목기준실패 시
비어 있지 않음 문자 수 > 0 즉시 실패
최소 길이 문자 수 ≥ 50 (기본값) 에스컬레이션
헤딩 보존 # 헤딩 1개 이상 경고만
반복 이상 10자+ 동일 문자열 5회+ 반복 에스컬레이션
추출 비율 텍스트 길이 / 원본 크기 ≥ 1% 에스컬레이션

🖼️ 이미지 OCR (PaddleOCR 무료)

PaddleOCR은 Baidu가 개발한 오픈소스 OCR 엔진입니다. OpenAI API 키 없이 로컬 PC에서 이미지 파일을 무료로 텍스트 추출할 수 있습니다. 한국어와 영문을 동시에 인식하며, GPU 없이 CPU만으로도 동작합니다.

🆓
이미지 파일은 자동으로 PaddleOCR 모드가 선택됩니다. 별도 설정 없이 .jpg, .png, .tif 등을 convert_one.py에 넘기기만 하면 됩니다.

패키지 설치 (최초 1회)

powershell 
pip install paddlepaddle paddleocr

⚙️ 설정 파일

모든 임곗값과 경로 설정은 config\settings.yaml에서 관리합니다.

config/settings.yaml 
# 경로 설정
paths:
  input: "input"
  raw_md: "output/raw_md"
  clean_md: "output/clean_md"
  json_logs: "output/json_logs"
  failed: "output/failed"

# 모드 설정
modes:
  default: "standard"
  chain: ["standard", "paddle_ocr", "ocr", "high_accuracy"]
  extension_overrides:
    ".tif": "paddle_ocr"   # 특정 확장자 모드 강제 지정
    ".tiff": "paddle_ocr"

# PaddleOCR 설정 (무료, 로컬 실행)
paddle_ocr:
  lang: "korean"        # 한국어+영문 동시 인식
  use_angle_cls: true   # 기울어진 텍스트 자동 보정

# 검증 임곗값
validation:
  min_char_count: 50         # 최소 문자 수
  max_repetition_count: 5    # 반복 허용 횟수
  min_extraction_ratio: 0.01 # 원본 대비 최소 추출 비율 (1%)

# 배치 처리
batch:
  max_workers: 4       # 병렬 워커 수
  continue_on_error: true

❓ FAQ

Q. 이미지 파일(.jpg, .png 등)을 변환하려면 어떻게 하나요?

이미지 파일은 PaddleOCR 모드로 자동 처리됩니다. 먼저 패키지를 설치한 뒤, convert_one.py에 이미지 경로를 넘기면 됩니다.

powershell 
# 1. 패키지 설치 (최초 1회)
pip install paddlepaddle paddleocr

# 2. 이미지 변환 (input 폴더 사용)
python scripts\convert_one.py input\scan.jpg

# 2-1. 또는 경로 직접 지정 (input 폴더 불필요)
python scripts\convert_one.py "C:\Users\user\Downloads\receipt.png"

Q. PaddleOCR 첫 실행 시 뭔가 많이 다운로드되는데 정상인가요?

정상입니다. 최초 1회에 한해 한국어 인식 모델(약 500MB~1GB)이 자동으로 다운로드됩니다. 인터넷 연결이 필요하며, 다운로드 완료 후에는 오프라인에서도 동작합니다. 두 번째 실행부터는 모델이 이미 있으므로 즉시 시작됩니다.

Q. 이미지 변환 속도가 느린데 정상인가요?

PaddleOCR은 CPU만으로 동작하기 때문에 이미지 크기에 따라 5~30초가 소요됩니다. A4 스캔 문서 기준 약 10~20초가 일반적입니다. GPU가 있다면 훨씬 빠르게 처리됩니다. 여러 이미지를 처리할 때는 배치 변환을 사용하면 병렬 처리가 됩니다:

powershell 
python scripts\convert_batch.py input\ --workers 2

단, PaddleOCR은 CPU 집약적이므로 워커 수를 너무 높이면 오히려 느려질 수 있습니다.

Q. "Could not get FontBBox" 메시지가 계속 나와요

PDF 내부의 폰트 메타데이터가 불완전할 때 pdfminer가 출력하는 경고입니다. 텍스트 추출에는 영향이 없으며 무시해도 됩니다. 변환은 정상적으로 진행됩니다.

Q. "Couldn't find ffmpeg" 경고가 뜨는데요

오디오 처리 라이브러리(pydub)가 출력하는 경고입니다. PDF·문서 변환에는 ffmpeg가 전혀 필요 없으므로 무시해도 됩니다. 오디오 파일을 변환할 때만 ffmpeg 설치가 필요합니다.

Q. 변환 결과가 마크다운 헤딩 없이 텍스트만 있어요

원본 PDF가 시각적 디자인(폰트 크기·색상)으로만 구조를 표현한 경우 발생합니다. PDF 내부에 구조 태그가 없으면 MarkItDown도 헤딩을 인식할 수 없습니다. Word(.docx)나 HTML처럼 구조 정보가 있는 파일은 헤딩이 잘 보존됩니다.

Q. 이미 변환된 파일을 다시 변환하고 싶어요

powershell 
python scripts\convert_one.py input\파일.pdf --force

Q. 변환 속도를 높이고 싶어요

powershell 
# 워커 수를 CPU 코어 수만큼 늘립니다
python scripts\convert_batch.py input\ --workers 8

또는 config\settings.yaml의 batch.max_workers 값을 변경하면 기본값이 적용됩니다.
